# Receipt Extractor - Backend API

License validation and management API for Receipt Extractor desktop application.

## 🚀 Quick Start

### Prerequisites
- Node.js 16+ installed
- MySQL 8.0+ installed and running
- npm or yarn package manager

### Installation

1. **Navigate to backend directory:**
   ```bash
   cd backend
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure environment:**
   ```bash
   cp .env.example .env
   ```
   
   Edit `.env` and update:
   - Database credentials (DB_USER, DB_PASSWORD, DB_NAME)
   - API keys (GROQ_API_KEY, OPENAI_API_KEY)
   - Admin API key (will be generated in next step)

4. **Initialize database:**
   ```bash
   npm run init-db
   ```
   
   This will:
   - Create the database and tables
   - Generate an admin API key
   - Display the admin key (save it to .env!)

5. **Generate license keys:**
   ```bash
   npm run generate-keys pro 100
   npm run generate-keys budget 100
   ```

6. **Start the server:**
   ```bash
   npm start
   ```
   
   For development with auto-reload:
   ```bash
   npm run dev
   ```

## 📡 API Endpoints

### Public Endpoints

#### Validate License
```http
POST /api/license/validate
Content-Type: application/json

{
  "license_key": "PRO-XXXX-XXXX-XXXX",
  "device_fingerprint": "unique-device-id",
  "device_name": "John's Laptop",
  "ip_address": "192.168.1.1"
}
```

**Response (Budget):**
```json
{
  "valid": true,
  "tier": "budget"
}
```

**Response (Pro):**
```json
{
  "valid": true,
  "tier": "pro",
  "api_keys": {
    "groq": "gsk_...",
    "openai": "sk-..."
  }
}
```

### Shopify Webhooks

#### Order Created
```http
POST /api/shopify/order-created
X-Shopify-Hmac-Sha256: <signature>
Content-Type: application/json

{
  "id": 12345,
  "order_number": 1001,
  "email": "customer@example.com",
  ...
}
```

### Admin Endpoints

All admin endpoints require header:
```
X-Admin-API-Key: your_admin_api_key_here
```

#### Get Statistics
```http
GET /api/admin/stats
```

#### Get All Licenses
```http
GET /api/admin/licenses?tier=pro&is_used=false&limit=50
```

#### Get License Details
```http
GET /api/admin/license/PRO-XXXX-XXXX-XXXX
```

#### Revoke License
```http
POST /api/admin/license/revoke
Content-Type: application/json

{
  "license_key": "PRO-XXXX-XXXX-XXXX",
  "reason": "Refund requested"
}
```

#### Remove Device Activation
```http
DELETE /api/admin/activation/123
```

## 🗄️ Database Schema

### Tables

- **license_keys** - All generated license keys
- **license_activations** - Device activations (max 2 per license)
- **shopify_orders** - Orders from Shopify
- **api_logs** - API request logs
- **admin_users** - Admin API keys

### Views

- **license_statistics** - Key usage stats by tier
- **activation_statistics** - Activation stats by tier

## 🔧 Configuration

### Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `PORT` | Server port | `3000` |
| `DB_HOST` | MySQL host | `localhost` |
| `DB_USER` | MySQL user | `receipt_admin` |
| `DB_PASSWORD` | MySQL password | `secure_password` |
| `DB_NAME` | Database name | `receipt_extractor` |
| `GROQ_API_KEY` | Groq API key (for Pro users) | `gsk_...` |
| `OPENAI_API_KEY` | OpenAI API key (for Pro users) | `sk-...` |
| `ADMIN_API_KEY` | Admin panel API key | Generated by init-db |
| `SHOPIFY_WEBHOOK_SECRET` | Shopify webhook secret | From Shopify |
| `MAX_ACTIVATIONS_PER_LICENSE` | Max devices per license | `2` |

## 🛒 Shopify Integration

### Setup Webhook

1. Go to Shopify Admin → Settings → Notifications
2. Create webhook:
   - Event: `Order creation`
   - Format: `JSON`
   - URL: `https://your-domain.com/api/shopify/order-created`
3. Copy webhook secret to `.env`

### Product Configuration

Set product titles or SKUs to include:
- "Pro" or "PRO" → Assigns Pro tier license
- "Budget" or "BUDGET" → Assigns Budget tier license

## 🚨 Troubleshooting

### Database Connection Failed
- Check MySQL is running: `mysql -u root -p`
- Verify credentials in `.env`
- Ensure database exists: `npm run init-db`

### No Available License Keys
- Generate more keys: `npm run generate-keys pro 100`
- Check available keys in admin stats

### Shopify Webhook Not Working
- Verify webhook secret in `.env`
- Check server logs for errors
- Test with Shopify webhook tester
